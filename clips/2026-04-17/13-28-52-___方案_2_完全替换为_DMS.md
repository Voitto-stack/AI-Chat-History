## 方案 2：完全替换为 DMS

### 核心思路

不再使用 Prisma，直接用 DMS 的 `getPool()` + raw SQL，完全复用 DMS 的拦截逻辑。

### 实现代码

```typescript
// apps/minerva-server/src/db/dms-pool.ts
import { getPoolForDatabase } from '@sitin/business-dms-datasource';
import { sqlExecute } from '@sitin/business-dms-sql';

export async function executeDMSQuery(
  userId: string,
  userRole: string,
  sql: string,
  params?: any[]
) {
  // 直接复用 DMS 的完整拦截逻辑
  return sqlExecute({
    userId,
    userRole,
    dataSourceId: 'minerva-prod-pg',
    database: 'sitin_db',
    sql,
    params
  });
}
```

### 使用方式

```typescript
// ❌ 不再用 Prisma
// await prisma.userinfo.findMany({ where: { user_id: 123 } });

// ✅ 改用 DMS 执行
const users = await executeDMSQuery(
  userId,
  userRole,
  'SELECT * FROM userinfo WHERE user_id = $1',
  [123]
);
```

### 方案 2 的问题

- **需要重写所有查询** — 工作量巨大
- **失去类型安全** — 没有 Prisma 的类型推导
- **SQL 手写容易出错** — 需要手动处理参数绑定
