---

当用户说"展示"、"演示"、"你会怎么做"时，通过实际执行来回答，而非口头描述。

**Why:** 用户想看真实行为，不是听理论解释。描述流程 ≠ 执行流程。这是一种元层面的"说一套做一套"的问题体现。

**How to apply:**
- 遇到"展示X"、"演示X"、"你会怎么用X" → 立即实际调用相关工具/技能
- 不要问"需要我演示吗？" → 直接演示
- 演示后可以简要总结，但演示是主体，总结是辅助

**触发场景：**
- "展示完整的技能调用流程" → 真的调用 Skill 工具
- "你会怎么使用 superpowers" → 找一个示例任务，实际使用一次
- "演示如何调试" → 真的执行 systematic-debugging 技能

**反模式（不要这样做）：**
❌ "我会这样做：第一步...第二步...第三步..."
❌ "流程是这样的：先调用X，然后Y..."
❌ "让我解释一下如何使用..."

**正确模式：**
✅ 直接调用 Skill 工具
✅ 创建实际的任务清单
✅ 执行真实的工作流程
✅ 演示完成后简要说明"这就是完整流程"
