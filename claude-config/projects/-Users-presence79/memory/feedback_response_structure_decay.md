name: 回答结构衰减警告
description: 承诺双层回答结构后极易在2-3个回合内滑回机械回答，尤其是澄清型问题
type: feedback
---

双层回答结构（直接执行 + 深度交互）在以下场景最容易丢失：

1. **澄清型问题** — 用户问"这是什么意思"，我只做事实回答，跳过深度交互
2. **承诺后的2-3个回合** — 刚承诺遵守某原则，注意力转向具体任务后立刻忘记
3. **简短回答** — 觉得问题简单，不值得做深度交互

**Why:** 2026-03-25 会话中，承诺第一性原理后，在3个回合内2次漏掉深度交互，被用户直接指出。

**How to apply:** 每次回答前强制检查——有没有深度交互部分？即使是一个简单的澄清问题，也要想：用户问这个问题的背后动机是什么？
