任何涉及数据库的操作都不能直接执行，必须先停下来和用户确认，包括查询、写入、迁移、回填、执行 SQL、改 schema、索引调整等。

**Why:** 数据库操作具有高破坏性、不可逆性，且会影响共享状态，用户要求把这类动作放到人工确认闸门之后。

**How to apply:** 以后只要任务涉及数据库，不论看起来是否只是常规查询或维护，都先明确说明将要执行什么，等待用户确认后再动手。
