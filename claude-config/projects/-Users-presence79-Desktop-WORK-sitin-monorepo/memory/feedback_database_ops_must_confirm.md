---
name: 数据库操作必须先确认
description: 涉及任何数据库增删改查操作时，必须先停下来获得用户明确许可，不可擅自操作
type: feedback
---

**规则：** 在编写代码过程中，如果遇到**任何**数据库相关的操作（包括增删改查、查询、schema 变更等），必须停下来与用户确认，获得明确许可后才能继续操作。

**Why:** 数据库操作具有持久性和潜在破坏性，即使是查询操作也可能涉及敏感数据、造成性能影响或暴露业务逻辑。用户需要对所有数据库访问保持完全的控制权和知情权。

**How to apply:**
- 当编写包含 Prisma Client 任何操作（create、update、delete、findMany、count、aggregate 等）时，立即暂停
- 当编写包含原始 SQL（SELECT、INSERT、UPDATE、DELETE 等）时，立即暂停
- 当编写数据库 schema 变更（migration、schema 文件修改）时，立即暂停
- 向用户说明将要执行的数据库操作、访问的表、查询条件和预期影响范围
- 等待用户明确的确认或许可
- **无例外**：无论读写、无论环境、无论表的敏感程度，一律先确认
