排查 bug 时，先直接说当前情况和修改方案，用几句话讲清楚，不要堆长篇解释、背景铺垫或文档式结构。

**Why:** 用户明确反馈“看不懂”“不要再写一堆没用的文档”，希望只听可执行结论。

**How to apply:** 以后在 Minerva 这类排查/修复对话里，优先给“现在什么情况 + 要怎么改”两句式结论；只有用户追问时再展开证据链。
