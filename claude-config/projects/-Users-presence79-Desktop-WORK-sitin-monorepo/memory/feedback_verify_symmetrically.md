description: UI 对比测试时，不能信 DOM 查询的完备性（虚拟滚动只渲染可见项），先读代码再看 UI，两端必须用相同方法验证
type: feedback
---

对比测试新老项目时，发现差异应先质疑测量方法是否一致，而非直接下结论。

**Why:** 在 screenshot-review 测试中，用 `querySelectorAll` 获取 Ant Design Select 的选项，因虚拟滚动只返回可见的 10 个。对新项目做了滚动二次验证发现了全部 14 个，但对老项目没做同样验证，导致误报"老项目只有10个选项"。

**How to apply:**
1. 不要信 DOM 查询作为数据完备性依据 — 虚拟滚动、懒加载会导致 DOM 只反映部分数据
2. 先读源码确认 source of truth，再用 UI 做交叉验证
3. 对比测试中，两端必须用完全相同的验证方法，发现差异时第一反应是检查测量方法是否对称
