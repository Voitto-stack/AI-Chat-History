## MCP Server 的标准实现模式

### 1. 基础结构（以 Discord/Fakechat 为例）

```typescript
import { Server } from '@modelcontextprotocol/sdk/server/index.js'
import { StdioServerTransport } from '@modelcontextprotocol/sdk/server/stdio.js'
import { ListToolsRequestSchema, CallToolRequestSchema } from '@modelcontextprotocol/sdk/types.js'

// 创建 MCP server 实例
const mcp = new Server(
  { name: 'my-server', version: '1.0.0' },
  {
    capabilities: { tools: {} },
    instructions: 'Optional system instructions for Claude'
  }
)

// 注册工具列表处理器
mcp.setRequestHandler(ListToolsRequestSchema, async () => ({
  tools: [
    { name: 'tool1', description: '...', inputSchema: { ... } },
    { name: 'tool2', description: '...', inputSchema: { ... } }
  ]
}))

// 注册工具调用处理器
mcp.setRequestHandler(CallToolRequestSchema, async (request) => {
  const { name, arguments } = request.params
  // 分发给具体的工具处理函数
  return { content: [{ type: 'text', text: result }] }
})

// 连接到 Claude
await mcp.connect(new StdioServerTransport())
```

### 2. 工具定义的关键字段

- **name**: 工具标识符
- **description**: 向 Claude 说明工具用途
- **inputSchema**: JSON Schema 格式的参数定义
  - properties: 参数及其类型
  - required: 必需参数列表
  - additionalProperties: 是否允许额外参数

### 3. 工具返回格式

```typescript
{
  content: [
    {
      type: 'text',  // 或 'image', 'resource'
      text: '结果内容'
    }
  ],
  isError?: boolean  // 可选，标记错误
}
```

## 关键设计模式

### 1. 代理模式（Proxy Pattern）
Claude-Mem 使用：MCP server 本身是一个薄包装，实际业务逻辑运行在独立的 Worker 进程
- 优点：业务逻辑和 MCP 协议处理分离，易于重启 Worker
- 通信：HTTP 长连接（socket 或 TCP）

### 2. 通知模式（Notification Pattern）
Discord/Fakechat 使用：MCP server 推送事件给 Claude
- 入站通知：`mcp.notification({ method: 'notifications/claude/channel', params })`
- 权限通知：`mcp.notification({ method: 'notifications/claude/channel/permission', params })`
- 用途：实现消息接收、权限决策流程

### 3. 状态文件模式（State File Pattern）
Discord 使用：在文件系统中存储配置和临时状态
- 配置：`~/.claude/channels/discord/access.json`
- 待决配对：`access.json` 中的 `pending` 字段
- 批准文件：`~/.claude/channels/discord/approved/<userId>`

### 4. 权限控制模式（Permission Control Pattern）
- 入站通道（Inbound Gate）：决定哪些消息被接受
- 出站通道（Outbound Gate）：检查工具调用目标是否被允许
- 示例：Discord 的 `gate()` 函数在工具调用前验证权限

## 关于包装多个 AI 模型的可能性

### 当前观察

1. **Replicate MCP** - 已在配置中
   - 封装 Replicate API 的模型调用能力
   - 通过工具接口暴露各类模型（LLM、图像生成等）

2. **Claude-Mem Worker** 调用 Claude Agent SDK
   - Worker 内部可调用多个 Claude 模型（不同版本）
   - 通过 HTTP 端点暴露给 MCP server

### 实现方向

要创建**多模型包装 MCP server**：

```typescript
const tools = [
  {
    name: 'call_gpt4',
    description: 'Call OpenAI GPT-4',
    inputSchema: { 
      type: 'object',
      properties: { prompt: { type: 'string' } },
      required: ['prompt']
    },
    handler: async (args) => {
      const result = await openai.chat.completions.create({
        model: 'gpt-4',
        messages: [{ role: 'user', content: args.prompt }]
      })
      return { content: [{ type: 'text', text: result.choices[0].message.content }] }
    }
  },
  {
    name: 'call_claude',
    description: 'Call Claude via Agent SDK',
    inputSchema: { /* ... */ },
    handler: async (args) => {
      const message = await claude.query(args.prompt)
      return { content: [{ type: 'text', text: message }] }
    }
  },
  // ... 更多模型
]
```

关键点：
- 每个模型调用都是一个工具
- 工具处理函数可以是异步的，支持网络调用
- 结果统一包装为 `{ content: [...] }` 格式

## 关键代码片段

### 工具分发的标准模式
```typescript
server.setRequestHandler(CallToolRequestSchema, async (request) => {
  const tool = tools.find(t => t.name === request.params.name)
  if (!tool) throw new Error(`Unknown tool: ${request.params.name}`)
  
  try {
    return await tool.handler(request.params.arguments || {})
  } catch (error) {
    return {
      content: [{ type: 'text', text: `Error: ${error.message}` }],
      isError: true
    }
  }
})
```

### HTTP 代理调用
```typescript
async function callWorkerAPI(endpoint: string, params: Record<string, any>) {
  const searchParams = new URLSearchParams()
  for (const [key, value] of Object.entries(params)) {
    if (value !== undefined && value !== null) {
      searchParams.append(key, String(value))
    }
  }
  const response = await workerHttpRequest(`${endpoint}?${searchParams}`)
  const data = await response.json()
  return { content: [{ type: 'text', text: JSON.stringify(data) }] }
}
```
