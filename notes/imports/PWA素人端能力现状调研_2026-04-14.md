## 二、注册与入驻流程

### 2.1 登录方式

- **Google OAuth** — 唯一正式登录方式。调用 `startGoogleOAuth()` 跳转 Google 授权页，隐式授权获取 `id_token`，解析出 `sub`(Google User ID) + `email`，调用后端 `FastLoginRequest`
- **Quick Login** — 仅非 production 环境显示，用 `deviceId` 快速登录（开发调试用）
- **无**手机号/短信验证码登录，**无**邮箱密码登录

**关键文件**：
- `src/pages/Login/index.tsx`
- `src/utils/googleAuth.ts`

### 2.2 Onboarding（4步注册）

登录后如果 `userInfo.pending === true`，路由自动跳转到 Onboarding 容器（`/onboardingcontainer`）。

| 步骤 | 组件 | 收集内容 | 奖励 | 门槛 |
|------|------|---------|------|------|
| 1 | `Username.tsx` | 用户名（2-32字符，字母数字._） | $0.1 | 无 |
| 2 | `Age.tsx` | 年龄（18-99） | $0.1 | 无 |
| 3 | `Photo.tsx` | 上传1张头像（含裁剪） | $0.3 | **颜值打分 ≥ 60**，年龄 > 46 拒绝 |
| 4 | `Phone.tsx` | 美国手机号（US+1） | 无 | 仅格式校验，无短信验证 |

最后提交 `uploadUserInfo()`（性别硬编码 `FEMALE`）+ `editProfile({ phoneNumber })`，完成 `TaskId.Register`。

**注意**：
- 无位置/地址收集（虽然 events.ts 定义了 location 事件，Onboarding 中未实现）
- 无性别选择（硬编码女性）
- 无兴趣标签选择（events.ts 有定义但未实现）
- 手机号仅支持美国号码

**关键文件**：
- `src/pages/Onboarding/index.tsx`（容器，`TOTAL_PAGES = 4`）
- `src/pages/Onboarding/Username.tsx`
- `src/pages/Onboarding/Age.tsx`
- `src/pages/Onboarding/Photo.tsx`
- `src/pages/Onboarding/Phone.tsx`

### 2.3 人脸验证（注册后独立步骤）

注册阶段**无**身份认证/活体检测。人脸验证是 `TaskId.FaceVerify = 118`，**提现前的必要条件**。

流程：上传照片 → 颜值打分（≥60）→ 旷视(Megvii) FaceID H5 活体检测 → 后端校验 → 标记 `isFaceVerified`

**关键文件**：
- `src/pages/VerifyPhoto/index.tsx`
- `src/hooks/useVerifyPhoto.ts`

### 2.4 路由守卫

- `ProtectedRoute`：未登录 → `/onboarding`；已登录未完成注册 → `/onboardingcontainer`
- `OnboardingRoute`：已完成注册 → 首页 `/`
- 只要用户打开 App 但未完成注册，强制拦截到 Onboarding 页
- **无主动推送/短信/邮件召回**

**关键文件**：`src/router/index.tsx`

## 三、首次上线与 Mock 通话机制

### 3.1 开播流程

1. 首页点击 "Go LIVE" → 3秒倒计时 → 调用 `changeDispatchStatus(true)` 通知后端可分发来电
2. 提示文案："Estimated 1-5 minutes to receive a call"

### 3.2 Mock 通话（核心机制）

**新用户的"首次通话"是 Mock 通话（预录视频），不是真实来电。**

- **StageTwo**：Go Live 后**立即触发** Mock 视频通话（$1.5），无需等待
- **第二次**：$0.6
- **StageThree+**：3分钟无真实来电自动触发补贴 Mock（$0.6/min）
- **Dispatch Detection**：非 Live 状态下，5-10s 后自动下发 Dispatch 类型 Mock 来电

Mock 通话页面（`/mock-call`）：上方本地摄像头，下方播放预录视频，有 NoFace/NoVoice 检测。

**挂断保护**：
- ≤28s 弹违规警告
- 28-30s 弹挽留
- ≥30s 正常确认
- 3次违规收益清零

**关键文件**：
- `src/pages/Live/PreLiveView.tsx`（Mock 触发逻辑）
- `src/pages/MockCall/index.tsx`
- `src/utils/mockCallManager.ts`
- `src/hooks/useMockCallScheduler.ts`
- `src/hooks/useDispatchDetection.ts`
- `src/components/ExitCallModal.tsx`（挂断保护）

### 3.3 真实通话

真实来电走 TRTC/TUICall SDK：
- 视频通话：`/video-call`（`webCallManager` 处理）
- 语音通话：`/audio-call`
- 来电通过 WebSocket `CALL_INVITED` 事件触发，前端自动 `accept()`
- Mock 和真实通话互斥：真实来电会自动取消 Mock

**关键文件**：
- `src/pages/VideoCall/index.tsx`
- `src/pages/AudioCall/index.tsx`

## 四、匹配机制

**前端无匹配逻辑，完全由后端控制，素人端是被动等待模式。**

- `changeDispatchStatus(true)`：告诉后端"我在线可接来电"
- 后端有**流量池分层**（通过埋点事件名推断）：
  - `join_hatching_pool` — 孵化池（新主播？）
  - `join_the_racing_pool` — 赛马池
  - `join_main_flow_pool` — 主流量池
  - `disuse_pool` — 淘汰池
- 具体匹配权重和分配策略在后端，前端不可见

**关键文件**：
- `src/http/liveApi.ts`（`changeDispatchStatus`）
- `src/tracking/events.ts`（流量池事件定义）

## 七、提现系统

### 7.1 支付方式

| 方式 | 状态 | 解锁条件 |
|------|------|---------|
| **PayPal** | 可用 | 默认 |
| Virtual Visa | 锁定 | 余额 ≥ $100 |
| Amazon | 锁定 | 余额 ≥ $100 |
| Bank (ACH) | 锁定 | 余额 ≥ $20 |

### 7.2 PayPal 绑定

- Stage 1-2：邮箱绑定（输入邮箱即可）
- Stage 3+：OAuth 验证（跳转 PayPal 登录页授权，回调到 `/paypalresult`）

### 7.3 提现流程

```
REMINDER → BIND_PAYPAL/LINK_PAYPAL → PAYPAL_BIND_RESULT → PROCESSING → SUCCESS
```

- 必须按阶段顺序完成，每阶段所有任务完成 + 余额足够才能提现
- 反欺诈：GPS 不在美国 + PayPal 不是美国账户 → ban 用户
- 到账提示："Funds will appear in your PayPal account within 15 minutes."

## 八、留存与召回机制

### 8.1 Firebase Push 通知

- 初始化 Firebase Messaging，获取 FCM Token 上传服务端
- Service Worker 处理通知点击，导航到指定 URL
- APK 环境由原生处理推送
- **基建已有，但缺后端定时触发逻辑（如"你已48小时未上线"）**

### 8.2 假未接来电回拨

- 进入 Live 页面时拉取"未接来电"列表（`getMissedCall`）
- 弹出 `MissedCallsModalStack`（全屏卡片堆叠弹窗）
- 点击"回拨"后展示 `VideoMatchModal`（15秒旋转等待动画）
- 注意：这是被动机制（用户回来后才触发），非主动推送

### 8.3 Dispatch Mock 来电

- 非 Live 状态下，5-10s 后自动下发 Dispatch 类型 Mock 来电（StageThree+ 才触发）
- 每次从后台切回前台都会重新触发

### 8.4 离线收益弹窗

- 后台超过1小时回来，查询 Instagram 消息收益 + idle 等待收益
- 弹窗："欢迎回来，你离开期间赚了 $X"

### 8.5 APK 下载提示

- StageTwo 完成后每天弹一次 APK 下载弹窗
- SecondEarn 完成后自动弹一次

**关键文件**：
- `src/services/firebase.ts`
- `src/service-worker.ts`
- `src/hooks/useMissCall.tsx`
- `src/stores/missCallStore.ts`
- `src/hooks/useDispatchDetection.ts`

## 十、能力缺失汇总

| 缺失类别 | 具体缺失 | 影响 |
|----------|---------|------|
| **数据可视化** | 无转化漏斗看板、无 Onboarding 各步流失率看板、无渠道成材率对比 | 埋点数据采集了但无人可看，无法定位漏损 |
| **主动召回** | 无未完成注册短信/Push 召回、无未上线主动推送（Firebase 基建有但缺触发逻辑） | 只有被动机制（用户回来才触发），流失用户无法拉回 |
| **A/B 测试** | 无 A/B 测试框架或 feature flag 系统 | 无法科学评估优化效果 |
| **收入实时感知** | 无每笔收入 Toast/Push、无实时累计、无预估收入 | 用户对"赚钱"感知不够即时 |
| **AI 诊断** | 无基于行为数据的 AI 分析 + 个性化建议 | 干预只有 Mock 来电，不够精准 |
| **新手教程** | `pwa_dh_tutorial_*` 12个事件已定义但 UI 未实现 | 新手引导仅靠任务清单，缺少直观教程 |
| **社交/社群** | 无 peer support、无 mentor 机制、无素人之间的互动 | 缺少社交黏性和经验传递 |
| **成材分析** | 无成材 vs 流失行为差异分析、无成材路径建模 | 无法用数据定义"好主播"特征 |
