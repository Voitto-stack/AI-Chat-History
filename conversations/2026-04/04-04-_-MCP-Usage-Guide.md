## webclaw 使用场景

**适用情况**：
- 静态 HTML 页面（博客、文档、新闻）
- 服务端渲染的内容（SSR）
- 公开可访问的页面（无需登录）
- 只需读取文本/结构化数据
- 不需要点击/交互

**典型场景**：
- 抓取博客文章/Medium/Substack
- 读取技术文档（非 SPA）
- 提取新闻内容
- 爬取产品页面的文本信息
- 读取 GitHub README（虽然 GitHub 是 SPA，但 README 部分可能可以直接抓取）

**工作原理**：
- 发送 HTTP 请求
- 获取 HTML 源码
- 解析 DOM 结构
- 返回文本内容

**优势**：
- ✅ 快速（无需启动浏览器）
- ✅ 低资源消耗
- ✅ 简单直接
