## 📂 数据库相关代码位置

### 1️⃣ **核心业务层** (`business-dms-*`)
每个业务域都包含三个核心文件：
- `{module}.sql.ts` — SQL 语句常量
- `{module}.model.ts` — 数据模型和数据库操作
- `{module}.service.ts` — 业务逻辑

```
business-dms-user/          # 用户管理
business-dms-audit/         # 审计日志
business-dms-datasource/    # 数据源管理
business-dms-approval/      # 审批流
business-dms-saved-sql/     # 保存的 SQL
```

### 2️⃣ **数据库迁移** (`app-dms-server/src/migrate/`)
- `migrations.ts` — 内嵌的 SQL 迁移脚本（TypeScript 常量）
- `runner.ts` — 迁移执行器（启动时自动运行）

### 3️⃣ **数据库插件** (`server-plugin-*`)
- `server-plugin-pg/` — PostgreSQL 连接池封装
- `server-plugin-mysql/` — MySQL 连接池封装
- `server-plugin-redis/` — Redis 客户端封装（可选缓存层）

## 🏗️ 数据库架构

### **技术栈**
| 组件 | 技术 | 用途 |
|------|------|------|
| DMS 自身数据库 | PostgreSQL (`pg`) | 存储用户、审批、审计日志等 |
| 目标数据库连接 | `mysql2` + `pg` | 用户通过 DMS 连接的外部数据库 |
| 缓存层（可选） | Redis (`ioredis`) | 保存的 SQL 等读多写少的数据 |

## ⚙️ **2. 配置环境变量**

### **后端（app-dms-server）**

在 `packages/app-dms-server/` 下创建 `.env` 文件：

```bash
# 必填
DATABASE_URL=postgresql://user:password@localhost:5432/dms_db

# 可选（有默认值）
PORT=3001
NODE_ENV=development
JWT_SECRET=dev-secret-change-in-production
ENCRYPTION_KEY=dev-encryption-key-32-bytes-long
REDIS_URL=redis://localhost:6379  # 可选，不配置会 fallback 到 DB
```

**⚠️ 注意**：
- `DATABASE_URL` 是 **必填** 的 PostgreSQL 连接串
- 生产环境必须替换 `JWT_SECRET` 和 `ENCRYPTION_KEY`
- 默认端口 3001

### **前端（app-dms-web）**

在 `packages/app-dms-web/` 下创建 `.env` 文件：

```bash
VITE_API_BASE_URL=http://localhost:3001
```

## 🐘 PostgreSQL 安装方式

你有 **3 种方式** 提供 PostgreSQL 数据库：

---

### **✅ 方式一：Docker（推荐，最简单）**

无需本地安装，一行命令启动：

```bash
docker run --name dms-postgres \
  -e POSTGRES_USER=dms \
  -e POSTGRES_PASSWORD=dms123 \
  -e POSTGRES_DB=dms_db \
  -p 5432:5432 \
  -d postgres:16-alpine
```

然后在 `.env` 中配置：
```bash
DATABASE_URL=postgresql://dms:dms123@localhost:5432/dms_db
```

**停止/启动数据库**：
```bash
docker stop dms-postgres    # 停止
docker start dms-postgres   # 启动
docker rm dms-postgres      # 删除容器
```

---

### **✅ 方式二：本地安装 PostgreSQL**

适合长期开发：

**macOS**:
```bash
brew install postgresql@16
brew services start postgresql@16
createdb dms_db
```

**Linux**:
```bash
sudo apt install postgresql-16  # Ubuntu/Debian
sudo systemctl start postgresql
sudo -u postgres createdb dms_db
```

---

### **✅ 方式三：云数据库（远程）**

使用远程 PostgreSQL 服务：
- Supabase（有免费额度）
- Neon（Serverless PostgreSQL）
- Railway
- 腾讯云 / 阿里云数据库

直接配置远程连接串：
```bash
DATABASE_URL=postgresql://user:pass@remote-host.com:5432/dms_db
```

---

## 📌 关键点

1. **不需要本地安装**，只要能提供一个可访问的 PostgreSQL 连接串即可
2. **不能用其他数据库替代**（MySQL、SQLite 不行），因为：
   - 代码使用 `pg` 包（PostgreSQL 专用）
   - Migration 使用了 PostgreSQL 特有语法（如 `gen_random_uuid()`）
3. **推荐 Docker 方式**：干净、隔离、一键启动/停止
