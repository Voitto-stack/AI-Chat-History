## 2. 深度交互

### 问题根源分析

**为什么会停止？**  
PM2 daemon 在某个时刻重启了（可能是系统重启、内存清理或手动操作），但没有配置开机自启，导致所有定时任务丢失。

**隐藏的脆弱性**：
1. **单点故障** - 所有任务都依赖 PM2，一旦 PM2 崩溃就全部失效
2. **缺乏监控** - 任务停止了3天（4月7日至今）才被发现
3. **网络不稳定** - 日志显示大量 DNS 解析失败和超时错误

### 更健壮的方案

**选项A：直接使用系统 cron（推荐）**  
- 优势：系统级守护，不依赖 Node.js 进程
- 实现：`crontab -e` 添加：
  ```cron
  3 8 * * * cd ~/.morning-brief && node morning-brief.js
  30 8 * * * cd ~/macro-daily && node macro-daily.js
  0 9 * * * cd ~/.cs-daily && node cs-daily.js
  3 12 * * * cd ~/.leetcode-daily && node leetcode-daily.js
  ```

**选项B：添加健康检查**  
- 在飞书创建一个"心跳监控"机器人
- 每个任务成功执行后发送心跳
- 如果超过2小时没心跳，触发告警

**选项C：使用云函数托管**  
- 迁移到 Vercel Cron / AWS Lambda / 飞书定时任务
- 彻底摆脱本地环境依赖
