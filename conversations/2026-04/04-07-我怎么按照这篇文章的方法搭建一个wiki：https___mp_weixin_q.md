## 直接执行：按文章方法搭建Wiki

这篇文章介绍的是 **Andrej Karpathy 的 Claudian 项目**——用AI自动构建和维护个人知识库。

### 核心架构（10分钟搭建）

```bash
你的项目/
├── raw/           # 原始材料（文章、笔记、PDF等）
├── wiki/          # AI生成的知识页面（用[[双向链接]]）
├── outputs/       # 各种输出
└── CLAUDE.md      # 给AI的规则说明
```

### 搭建步骤

1. **安装 agent-browser**（可选，用于自动抓取网页）：
```bash
npm install -g agent-browser
agent-browser install
```

2. **创建目录结构**：
```bash
mkdir my-wiki && cd my-wiki
mkdir raw wiki outputs
```

3. **编写 CLAUDE.md**（核心！）：
```markdown
# 知识库说明
## 这是什么
一个关于[你的主题]的个人知识库

## 组织规则
- 每个主题一个 .md 文件
- 每个页面开头一段话总结
- 使用 [[双向链接]] 连接相关概念
- 在 INDEX.md 维护总览

## 新材料处理流程
1. 读懂材料核心观点
2. 在 wiki/ 创建/更新相关页面
3. 建立页面间链接
4. 标注新旧信息矛盾
5. 更新 INDEX.md
```

4. **把材料放入 raw/**

5. **在 Obsidian 中打开项目，用 Claude（Claudian插件）执行**：
   - "读完 raw/ 里的所有内容，按照 CLAUDE.md 的规则，编译一份wiki"

### 三个核心操作
- **Ingest**（摄入）：AI读材料 → 生成/更新wiki页面 → 建立链接
- **Query**（查询）：直接问问题，AI基于wiki回答
- **Lint**（体检）：AI检查矛盾、过时信息、孤儿页面

### 🚩 三个根本性问题

**1. 你真的需要"个人Wiki"吗？**
- **真实需求是什么**？快速查找？知识整理？写作素材？学习某个领域？
- 如果只是"收藏癖"→ Wiki会变成另一个信息坟墓
- 如果是学习 → 费曼笔记法（输出导向）比Wiki更有效

- 需要持续维护 CLAUDE.md（规则会过时）
