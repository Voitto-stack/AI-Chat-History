# 深度交互

## 根本原因分析

这不是配置错误，而是**依赖包生命周期问题**：

1. 原作者 `@cso1z` 可能删除了包或转移了维护权
2. 你的配置指向了一个已消失的包
3. MCP 服务器启动失败，但错误被静默吞掉

## 更优雅的方案

**建议采用版本锁定策略**：

```json
"args": ["feishu-mcp@0.2.7"]  // 锁定版本而非 @latest
```

**原因**：
- `@latest` 会自动拉取新版本，可能引入破坏性变更
- 锁定版本确保稳定性，升级时主动测试

## 预防措施

在 `~/.claude/` 创建监控脚本：

```bash
#!/bin/bash
# check-mcp-health.sh
for pkg in $(jq -r '.mcpServers[].args[]' ~/.mcp.json | grep -v '^-'); do
  npm view "$pkg" version 2>/dev/null || echo "❌ $pkg not found"
done
```

定期运行可提前发现包失效问题。
