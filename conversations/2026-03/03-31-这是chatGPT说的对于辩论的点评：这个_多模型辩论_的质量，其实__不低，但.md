## 直接回答：为什么没有事实库但模型还在引用 [F1]

**这是你的 `buildFactBank()` 的一个 bug。**

看代码 `battle-prompts.js:36-50`：

```javascript
const lines = userTask.split('\n');
for (const line of lines) {
    const trimmed = line.trim();
    if (trimmed.length > 10 && (
      /[\d%$￥]/.test(trimmed) ||   // ← 包含数字就命中
      /[：:]/.test(trimmed) ||       // ← 包含冒号就命中
      ...
    ))
```

你的"谁最菜"的 prompt 是：

> "你们**8**个模型...规则**：**1) 每个模型必须点名攻击你认为最弱的**1-2**个模型"

这一整行同时命中了 `\d`（数字8、1、2）和 `：`（冒号），所以**整个 prompt 被当成了 [F1]**。

结果就是：
- `facts = ["你们8个模型...投票选出谁是最菜的模型。规则：1) ..."]`
- 模型收到的事实库：`[F1] 你们8个模型...`
- 所以 F1 = **题目本身**

这就解释了你看到的现象：

| 模型 | 引用方式 | 是否合理 |
|------|---------|---------|
| Claude-Opus-4.6 | "根据题目规则[F1]" | 合理——F1 确实就是题目 |
| Claude-Sonnet-4.6 | "在这8个模型中[F1]" | 合理 |
| Claude-Opus-4.6 | "事实库中不包含任何基准测试数据[F1]" | 合理——诚实承认 |
| **Kimi-K2** | "训练语料70%以上截取自GitHub[F1]" | **造假**——F1里根本没这个 |
| **Kimi-K2** | "面对非代码题常转成伪代码卡死[F1]" | **造假**——把编造的攻击伪装成引用 |
| **Qwen3-Max** | "主谓宾都能错位的烂底子[F1]" | **造假**——把推断当事实引 |

**核心问题是两层的：**

1. **代码层 bug**：`buildFactBank()` 把用户 prompt 当事实提取了——只要包含数字或冒号就命中。应该先把 `userTask` 本身排除掉，只提取真正的数据行。

2. **模型层问题**：即便 F1 被定义为题目本身，Kimi-K2 仍然把自己编的攻击标上 [F1]，实质是**用引用格式伪装编造内容**。这证实了 ChatGPT 说的"伪事实引用"——但根因不只是模型胡编，你的系统也在帮倒忙。
