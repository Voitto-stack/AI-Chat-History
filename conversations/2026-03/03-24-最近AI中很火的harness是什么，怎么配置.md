## 什么是 AI Harness

**Harness** 在 AI 领域指的是**AI 代理的运行环境和控制框架**，它负责：

1. **工具调用管理** - 控制 AI 能使用哪些工具（文件操作、命令执行等）
2. **权限控制** - 定义哪些操作需要用户确认，哪些可以自动执行
3. **生命周期管理** - 处理会话启动、结束、钩子函数等
4. **环境配置** - 环境变量、模型选择、工作目录等

简单说：**Harness = AI 的"操作系统"**，决定 AI 能做什么、怎么做。

Claude Code 的 harness 通过 `settings.json` 配置：

```json
{
  // 1. 环境变量 - API密钥、服务配置
  "env": {
    "ANTHROPIC_AUTH_TOKEN": "sk-xxx",
    "FEISHU_APP_ID": "xxx"
  },

  // 2. 模型选择
  "model": "opus[1m]",  // opus 模型，1分钟超时
  
  // 3. 钩子函数 - 自动化行为
  "hooks": {
    "SessionStart": [...],  // 会话启动时执行
    "Stop": [...],          // 会话结束时执行
    "PreToolUse": [...],    // 工具调用前执行
    "PostToolUse": [...]    // 工具调用后执行
  },

  // 4. 权限配置
  "permissions": {
    "alwaysAllow": ["Bash:npm install"],
    "neverAllow": ["Bash:rm -rf /"]
  }
}
```

### 常用配置场景

**自动保存记忆（你正在用）：**
```json

  "Stop": [{
    "hooks": [{
      "type": "command",
      "command": "node sync-to-memory.js"
    }]
  }]
}
```

**工具使用监控：**
```json
"hooks": {
  "PostToolUse": [{
    "matcher": "Bash|Edit|Write",  // 匹配特定工具
    "hooks": [...]
  }]
}
```

核心原因：**从"对话 AI"到"自主 Agent"的范式转变**

1. **Agent 需要"围栏"** - AI 能自主使用工具后，需要精细控制边界

3. **工作流自动化** - hooks 机制让 AI 能触发外部系统（CI/CD、监控等）

- 早报系统是你的"生产环境"，每天自动运行
