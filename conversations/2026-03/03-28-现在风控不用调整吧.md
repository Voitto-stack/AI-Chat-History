### Assistant (11:23:17)

现有实现**已经自动处理了这个问题**，不需要额外调整。

## 原理说明

Firebase FCM 有天然的前台/后台分离机制：

| 状态 | 走哪个处理器 | 会弹推送？ |
|------|------------|---------|
| App 在前台（用户正在使用） | 无 `onMessage` handler → 消息静默丢弃 | **不会** |
| App 在后台 / 已关闭 | `firebase-messaging-sw.js` 的 `onBackgroundMessage` | **会** |

当前代码：
- `firebase.ts` 只注册了 token，**没有** `onMessage()` 前台监听
- `firebase-messaging-sw.js:27` 的 `onBackgroundMessage` 只在后台才触发

所以用户打开 App 时，FCM 消息会到达但没人处理它，**不会弹任何系统通知**。这是 Firebase 的默认行为，代码层面无需改动。

---

**深度交互**

如果服务端关心"不要给在线用户发推送"（节省 FCM 配额），那需要服务端感知用户在线状态，在下发推送前过滤掉在线用户——这是服务端的事，和客户端无关。你们有这个需求吗？

FCM 相关代码分布在 4 个文件：

```
apps/pwa/
├── public/
│   └── firebase-messaging-sw.js          # 后台推送处理（SW 环境）
└── src/
    ├── services/
    │   └── firebase.ts                    # Firebase 核心服务
    ├── hooks/
    │   ├── useFirebaseMessaging.ts        # 初始化入口（setup 函数）
    │   └── useUserInit.ts:93              # 调用点（登录后触发）
    └── service-worker.ts                  # 通知点击跳转处理
```

**各文件职责：**

| 文件 | 职责 |
|------|------|
| `firebase.ts` | Firebase 初始化、获取 FCM Token、上传 Token 到服务器 |
| `useFirebaseMessaging.ts` | 封装初始化流程（权限检查 + 重试逻辑） |
| `useUserInit.ts:93` | 用户登录后调用 `setupFirebaseMessaging()` |
| `firebase-messaging-sw.js` | 后台收到推送 → 显示系统通知 + 处理点击跳转 |
| `service-worker.ts` | 主 SW 的通知点击处理（与上面有重叠，`notificationclick` 事件） |

**数据流：**
```
用户登录
  → setupFirebaseMessaging()
  → initFirebase() + 权限检查
  → fetchFcmToken() → uploadFcmToken() → 服务器
  
服务器下发推送
  → 后台: firebase-messaging-sw.js → showNotification()
  → 前台: 无处理，静默丢弃
```

});

## 它的作用

`onMessage` 是 Firebase 提供的**前台消息监听器**：

| | `onMessage` | `onBackgroundMessage`（SW里） |
|--|--|--|
| 触发时机 | App 在**前台**时收到 FCM 消息 | App 在**后台/关闭**时 |
| 运行环境 | 页面 JS（window context） | Service Worker |
| 默认行为 | 不弹通知，只给你数据 | 自动弹系统通知 |
| 当前项目 | **未使用** | 已用（`firebase-messaging-sw.js:27`） |

## 当前项目里的导入情况

`firebase.ts` 只导入了 `getMessaging` 和 `getToken`，没有导入 `onMessage`：

```ts
import { getMessaging, getToken, isSupported, Messaging } from "firebase/messaging";
// onMessage 未导入 → 前台消息无人处理
```

所以前台收到的 FCM 消息会被 SDK 内部接收但没有 handler，直接丢弃。
